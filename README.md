# 🏬 MallPulse

**AI Co-Pilot for Shopping Mall General Managers**

MallPulse is a multi-agent AI system that turns raw retail data into instant, actionable intelligence. Mall GMs can ask natural-language questions about tenant performance, revenue trends, lease health, weather impact, and 30-day revenue forecasts — and get data-backed answers in seconds.

**🔗 Live demo:** https://mallpulse-3f3swnt3qq-uc.a.run.app

---

## What it does

| Ask this… | MallPulse answers with… |
|---|---|
| *"Who are the top 5 tenants at Forum Istanbul by revenue?"* | Ranked table with ₺ amounts and YoY change |
| *"Which tenants have leases expiring in the next 6 months?"* | Risk-flagged list with rent-to-sales ratios |
| *"Compare Zara across all 10 malls"* | Per-location revenue breakdown |
| *"What was the rain impact on Kanyon foot traffic in 2022?"* | Correlation table across weather buckets |
| *"Forecast next 30 days revenue for Kanyon"* | ARIMA-Plus daily forecast with confidence range |
| *"Is the Fivetran data pipeline healthy?"* | Live sync status from Fivetran API |

## Tech stack

| Layer | Technology |
|---|---|
| **AI agents** | Google ADK 1.34, Gemini 2.5 Flash (Vertex AI) |
| **Orchestration** | Root agent → 3 specialist sub-agents via `AgentTool` |
| **Pipeline tools** | Fivetran MCP server (live pipeline monitoring) |
| **Data warehouse** | BigQuery (`mallpulse_core` dataset, 8 tables) |
| **ML forecasting** | BigQuery ML `ARIMA_PLUS` (30-day revenue forecasts) |
| **Source database** | Cloud SQL Postgres (synced by Fivetran) |
| **UI** | Streamlit (dark theme, live tool-call status) |
| **Dashboard** | Looker Studio (revenue trends, tenant bar chart) |
| **Deployment** | Cloud Run (container, port 8080) |
| **Container registry** | Artifact Registry |

## Dataset

- **Source:** Istanbul Mall Customer Shopping dataset (Kaggle)
- **Coverage:** 10 Istanbul malls, January 2020 – March 2023
- **Volume:** ~99,458 transactions
- **Augmentation:** Synthetic tenant names, lease terms, Open-Meteo weather, Turkish public holidays — all generated by `prep_data.py`

## Three specialist agents

### 1 · Data Unifier
Retrieves raw data from BigQuery and monitors the Fivetran pipeline.
- Tools: `query_warehouse`, `get_mall_summary`, `get_weather_traffic_correlation`
- MCP: Fivetran REST API (connector health, last sync time, schema config)

### 2 · Tenant Diagnoser
Flags at-risk tenants and surfaces lease + revenue signals.
- Classifies tenants: 🔴 Critical / 🟡 Watch / 🟢 Healthy
- Rent-to-sales ratio benchmarks by tenant format (kiosk, inline, anchor, etc.)
- Tools: `query_warehouse`, `get_top_tenants`

### 3 · Action Recommender
Translates data insights into a prioritised GM action list.
- Three tiers: Immediate (this week) / Short-term (1–3 months) / Strategic (6–12 months)
- Every recommendation is data-backed and cites a forecast or ratio
- Tools: `query_warehouse`, `get_top_tenants`, `forecast_mall_revenue`

## Running locally

```bash
git clone https://github.com/heemaniar/mallpulse.git
cd mallpulse

python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# Fill in .env — BigQuery project, Postgres credentials, Fivetran API key

streamlit run app.py
# → http://localhost:8501
```

**Prerequisites:**
- GCP project with BigQuery enabled and `mallpulse_core` dataset loaded
- `gcloud auth application-default login` (for BigQuery + Vertex AI access)
- Postgres database populated via `python load_postgres.py`
- BigQuery loaded via `python load_bigquery.py`

## Deploying to Cloud Run

```bash
# One command — builds with Cloud Build, deploys to Cloud Run
bash deploy_cloudrun.sh
```

The script:
1. Creates an Artifact Registry repo (`mallpulse-repo`)
2. Builds the Docker image via Cloud Build (no local Docker needed)
3. Deploys to Cloud Run with all env vars from `.env`
